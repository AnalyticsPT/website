Measuring Iframes

What is an iframe?

An iframe, or inline frame, is an HTML element that allows you to embed another HTML document within a webpage. It is often used for forms and can be checked by right clicking on the form and finding the view frame source option. If this option doesnt appear, then it isnt an iframe.

What is the problem?

An iframe is basically a page within a page, and these two pages are completely independent. That means a GTM firing on the main page cant access the events within the iframe, and vice-versa.

One important step is to check the domain of the iframe by clicking on View frame source. If the iframe shares the domain of the main page then the implementation is simpler, if it doesnt then it is more complex.

Same Domain Implementation

If it is the same domain as the main website, then cookies are shared between both pages. This means that any event/conversion inside the iframe can also read the first party cookies from google analytics, google ads, facebook, etc. and attribute correctly to the same session/user.

To measure the events we just need to have the GTM implemented inside the iframe as well.

Once we have the GTM implemented in the iframe, then we will see both the main page and the iframe page in the preview. The iframe is treated as just another page, and we can implement events inside the iframe page as usual:

 

Different Domain Implementation

If the domain of the iframe is different from the main domain, then cookies arent shared between the two pages. Even if GTM is implemented, the tags inside the iframe dont have access to the analytics and advertising first party cookies, which means tags wont know its the same user and attribution will break.

It is possible to let iframes communicate with the main page using the javascript method postMessage (https://developer.mozilla.org/en-US/docs/Web/API/Window/postMessage).

In practical terms, we use javascript to send the event data from the iframe to the main page, and fire the event in the main page instead.

Send events from the iframe

To send event data from the iframe to main page we can use the following custom HTML tag. We just need to specify the domain that will receive this message, and send the information we want added to the datalayer. This tag should fire inside the iframe when the event happens:

<script>

function sendDataIframe(eventData) {

try {

var postObject = JSON.stringify(eventData);

parent.postMessage(postObject, 'https://www.dominioprincipal.pt/');

} catch (e) {

window.console && window.console.log(e);

}

}

 

sendDataIframe({

event: 'iframeEvent',

variable1: 'iframeVariable'

});

</script>

 

If the client is implementing a datalayer inside the iframe, then we can ask them to use the above code instead of the datalayer.push to simplify the implementation.

Receive events from the Iframe

In the main page, we then use this tag to read data from the iframe, and create the datalayer event. In this tag we just need to specify the domain of the iframe and it will automatically read all messages and create the datalayer event. This tag should be fired when the page loads, on the pages that have the iframe.

<script>

try {

window.addEventListener("message", function(message) {

 

if (event.origin !== "https://www.dominiodoiframe.pt/") return;

if (message.data != null && typeof message.data == 'object') return;

 

//console.log("event received!");

var data = JSON.parse(message.data);

 

var dataLayer = window.dataLayer || (window.dataLayer = []);

if (data.event) {

dataLayer.push(data);

}

});

} catch (e) {

window.console && window.console.log(e);

}

</script>

 

Without adding GTM or javascript to the iframe

If the client cant add GTM or javascript to the iframe, then we cant measure any events.
